Miro

Thinking

The team agreed to use a tool called Miro for the purposes of tracking ideas, design thoughts, high level requirements and general ideas. We used this as a base to elicit functional and non-functional requirements which were then recorded in a seperate tool. It also allowed us to work together in real time without the need for meetings while giving us a platform to review efforts on a regular basis.

Trello

Acting

The team agreed to use Trellio which is an Atlassian product which gave us the use of a board similar to a Jira scrum or agile board. This allowed us to capture detailed requirements and tasks for individuals. This would support the develoment process by the team been able to progress development through in progress to done in a typical sprint fashion.

Github

Doing

The team agreed to use a single Github repository for the code base for the development project/activities. By using a shared repository we were able to independently work on development activities.
