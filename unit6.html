Question 3

How does Flake differ from Pylint?

The exercise was to run and use flake8 against several .py files and to assess the difference between flake and pylint. The obvious difference is that flake looks more at style errors (it will look at syntax as well) so would be better suited for identifying incorrect indentation and line spacing for example. For example on the pylint_test.py file flake8 returned an error on line 16 to indicate an error on whitespace around operators. In the example below i am using the pylinttest.py script as previously and it is highlighting issues with whitespace.

Example Code

"""SOURCE OF CODE: https://docs.pylint.org/en/1.6.0/tutorial.html"""

import string

SHIFT = 3
choice = input("would you like to encode or decode?")
word = input("Please enter text")
LETTERS = string.ascii_letters + string.punctuation + string.digits

        if letter == ' ':
            ENCODED = ENCODED + ' '
        else:
            X = LETTERS.index(letter) + SHIFT
    ENCODED=ENCODED + LETTERS[X]
    if choice == "decode":
        for letter in word:
            if letter == ' ':
                ENCODED = ENCODED + ' '
            else:
                X = LETTERS.index(letter) - SHIFT
        ENCODED = ENCODED + LETTERS[X]

print(ENCODED)

Question 4

How does McCabe differ from other tools?

The exercise was to run and use mccabe against 2 pieces of code to understand the cyclomatic complexity of each. The results below show an example of the code been run and mccabe been executed against each one. The scripts themselves from a code perspective have very low cyclomatic complexity. It is also worth noting that mccabe comes as a built in lbrary with flake8 so additional commands were run. The conclusion is mccabe is suefule for determining cyclomatic complexity however the scripts used in the exercise have little to no complexity therefore score low.

# SOURCE OF CODE: https://realpython.com/python-testing/
