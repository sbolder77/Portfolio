What is a Secure Programming Language?

We were asked to answer a series of questions related to Python and secure programming languages and then share them with the team for discussion.


Question 1

What factors determine whether a programming language is secure or not?

There are several factors to consider when determining the secure nature of a programming language as well as considering known vulnerabilities. Cifuentes & Bierman (2019) discuss the three categories of vulnerability - buffers, injections, and information leaks. In addition to these considerations must be made to the number of vulnerabilities in the CWE (Common Weakness Enumeration) list. One must also consider the robustness and resiliency of a programming language. In addition, considerations should be made to the libraries used by a programming language and whether they can or cannot be used to secure methods such as authentication, encryption, and its interaction with and transaction of data.

Question 2

Could Python be classed as a secure language?

As with many programming languages, several factors should be taken into consideration. Python as a language does have the ability and libraries to perform secure coding practices but it is very dependent on the quality of code and the skill and experience of the developer. If a developer is inexperienced then regardless of what you program software to do, then carelessness will introduce vulnerabilities. Similarly, to using open source libraries without care and consideration to the authors and contributors to those libraries. A skilled developer would use experience and knowledge to write logical and well-structured code. khoirim (2020. Discuss many uses, advantages and disadvantages of Python. Whilst the points on advantages and disadvantages do not clearly call out if Python is a suitable language for secure programming the points raised do indicate it has limitations. However, dependent on the use of Python for specific tasks would point to Python having enough secure abilities. An assessment could be made at the point of design or analysis to use Python for the tactile development of a specific solution and the use of its libraries as well as the abilities of the developers to make it secure.

Question 3

Python would be a better language to create operating systems than C?

Given the points discussed and limited knowledge on C my initial thoughts would be that Python would not be a suitable or more appropriate language to create operating systems. Whilst Python has many advantages regarding simplicity it has disadvantages. Khoirim, Sonia, Laikhuram, Laishram & Singh (2020) highlight facts including the simplicity of the language, its limitation to 'one thread at a time' and limitations in database access as well as its limitations on memory comprehensive work. Considering Python as more of an interpreted language suitable to the application layer it perhaps would not be suitable for high-performance, multi-threaded and memory intensive computing tasks.

Producer Consumer

The exercise was run in the Codio environment. By having a prepared script, the code could be executed to output a result. While the exercise was simplistic in its nature to describe the use of queues and threads there were elements that could be improved.

Question 1

How is the queue data structure used to achieve the purpose of the code?

It supplies a linear data structure like a stack. As it stores items sequentially the consumer can then take the next item in the queue.

Question 2

What is the purpose of q.put(I)?

q.put[i] will add an element to the queue as well as a block if a queue has reached its maximum capacity.

Question 3

What is achieved by q.get()?

q.get() will remove and return an item from the queue.

Question 4

What functionality is provided by q.join()?

q.join() will block the threads until the consumers have processed the elements of the queue.

Question 5

What technique(s) would be appropriate to secure the code?

While not extending the code in this exercise there are several options to securing producer-consumer code. (Brownlee 2022) discusses the use of a signal shutdown where a sentinel object could be placed into the buffer by a producer acting as a clear message to consumers. This could prevent rogue input into a buffer potentially adding more elements. Other methods could be the use of max size parameters to limit the size of a queue. By setting this and adding logic in your code that controls how many elements are added to the queue you could restrict the risk of the incorrectly sized queue or making it infinite leading to overflow or corruption of data.

Code

'''code source: https://techmonger.github.io/55/producer-consumer-python/ - 
Module provising producer consumer printing function'''

from threading import Thread
from queue import Queue

q = Queue()
final_results = []

'''Function to calcualte the producer consumer threads'''
def producer():
    for P in range(100):
        q.put(P)        

def consumer():
    while True:
        number = q.get()
        result = (number, number**2)
        final_results.append(result)
        q.task_done()

for i in range(5):
    t = Thread(target=consumer)
    t.daemon = True
    t.start()

producer()
q.join()
print (final_results)
