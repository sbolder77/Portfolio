Faceted Data

In regard to Schmitz et al (2016) I think the approach is good but does have risks. The approach does have some benefits as described in the pros below. By been able to run multiple executions you can see better performance but at the financial risk of more computing resource. And while a library would be more usable and require less time on development there is a dependency to ensure that any such library is also maintained and updated. However, there are many additional factors to consider like testing and validation. With potentially complex mathematics and algortithims encapsulated in a library it would be difficult to make an assessment on it's use in secure software development.

Pros

- Information flow analysis enforces security in a systemic fashion
- Utilising non-interference ensures public inputs do not rely on private inputs
- Multiple executions within a single process with values containing public and private views of data
- Language runtime requires no modification and can be incorporated into existing language as a library

Cons

- Dependency on data and filesystems having adeqaute security in place
- Potentially difficult to analyze bugs
- Requires advanced mathematical knowledge
- Difficult to understand
- Has a high chance of becoming complex and risks introducing errors
- Higher performance on infrastructure

References:

Schmitz, T., Rhodes, D., Austin, T.H., Knowles, K. and Flanagan, C. (2016) Faceted dynamic information flow via control and data monads. In International Conference on Principles of Security and Trust: 3-23. Berlin, Heidelberg: Springer Berlin Heidelberg.
