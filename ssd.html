Secure Software Development

Formative Activities - Coding, Seminars & Team Activities

The module covered a range of formative assignments icluding coding concepts and seminar activities to support the objectives and delivery of a team development assignment. The content covered a wide range of concepts all of which were developed through the Codio IDE or independently in Visual Studio. Most of the evaluation and prototyping was carried out in Visual Studio to utilise a more familiar development environment.

Unit 1-3 Discussion 1

The activity was to write a discussive forum entry using a UML activity diagram and it's use to identify one of the OWASP top 10 weaknesses. Posts were reviewed by peers and included responses as well as a summary post.

Unit 2 - Scrum security review

The activity was to answer questions and discuss the software development stages of the scrum methodology. Additional activities were to discuss 5 items from the ISO/IEC 27000 standards and resource management in realtion to cyber security.

  • View Activity

Unit 3 - Team Discussion: What is a secure programming language?

The team were asked to work on a collaborative piece related to the determining factors of if a programming language is secure or not, Is Pythin secure and would python be better than C to create an OS.

  • View Discussion

Unit 3 - Exploring Python tools and features

In unit 3 we covered the concepts of buffers in both c and python. Using a combination of development tools like codio and Visual Studio the exercise was to assess the ouput form both c and pyhon buffers with elements of testing to identify and evaluate issues.

  • View Activity

Unit 4 - Programming language concepts

The seminar was based on a discussion around ReDOS and the use of 'Evil Regex'. We were asked to discuss the definition, common problems and mitigation with regex and how regex could be used in a secure solution

  • View Activity

Unit 5 - Equivalence Testing in Python

This coding activity was to execute and analyse an example of equivalence in Python. We had to consider how a partitioned set of objects can be evaluated with additional investigations in the code.

Unit 6 - Exploring Linters

The activity was to execute and investigate the use of linters in Python for the purposes of supporting testing. We had to execute several pieces of code utilising different linter libraries and provide an assessment piece.

Unit 7 - Exploring Python tools and features

Additional coding activites were supplied to investigate the use of a Python shell and also to implement a simple REST API for a distributed environment to be used fro creating and deleting records.

Unit 7 - What is an Ontology?

We had to provide a summary on our understanding of an ontology from our reading activities. We were also asked to define an ontology that would be relevant to our summative development assignment.

Unit 8 - Cryptography

The activity was to select a method of encryption and then write a Python program to encrypt a string value. Additionally had to create a Python program to make an encrypted copy of a text file.

Unit 8-10 Discussion 2

The activity was to design and present an ontology design capturing the weaknesses of TrueCrypt. Posts were reviewed by peers and included responses as well as a summary post.

Unit 10 - Faceted Data

We had to write a review of the use of faceted data and if a good approcah to protecting system from data leaks as well as create a basic design to create a system in Python.

Unit 11 - Microservices and Microkernels
